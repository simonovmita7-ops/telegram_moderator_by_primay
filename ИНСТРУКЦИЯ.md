## ⚡ Быстрый старт

### 1. Установка зависимостей

```bash
cd telegram-moderator-bot
python -m venv venv

# Windows:
venv\Scripts\activate
# Linux/Mac:
source venv/bin/activate

pip install -r requirements.txt
```

### 2. Настройка .env

Откройте файл `.env` и заполните:

```env
TELEGRAM_BOT_TOKEN=ВАШ_ТОКЕН_ОТ_BOTFATHER
AI_PROVIDER=gemini           # gemini / openai / claude
GEMINI_API_KEY=ВАШ_КЛЮЧ     # если используете Gemini
OPENAI_API_KEY=ВАШ_КЛЮЧ     # если используете OpenAI
```

**Где получить токен:** напишите @BotFather в Telegram → /newbot

### 3. Запуск бота

```bash
python -m bot.main
```

## 📋 Правила — правила.txt

### Как работает авто-обновление

Файл `правила.txt` в корне проекта — **единственный источник правил**.

- Бот читает файл при каждой проверке сообщения
- Если файл изменился — автоматически перезагружает его
- **Не нужно перезапускать бота!** Просто сохраните файл

### Формат файла

```
📋 ПРАВИЛА ГРУППЫ

1. НАЗВАНИЕ ПРАВИЛА
Описание того, что запрещено.

2. ДРУГОЕ ПРАВИЛО
Описание.
```

Бот передаёт весь текст файла ИИ как системные правила, и ИИ работает **строго по ним**.

## 🔤 Управление словами

### Запрещённые слова (немедленная реакция)

Бот реагирует мгновенно при обнаружении слова, **без ИИ**:

```
/addword слово    — добавить запрещённое слово
/delword слово    — удалить запрещённое слово
```

Также доступно в панели группы → **Настройки → Запрещённые слова**

### Слова-исключения (бот игнорирует)

Если слово находится в списке исключений — бот **не реагирует** на него вообще:

```
/addexc слово     — добавить исключение
/delexc слово     — удалить исключение
```

Также доступно в панели группы → **Настройки → Слова-исключения**

## 🔔 Проверка подписки на канал

### Как настроить

1. Откройте бота в личных сообщениях
2. Нажмите **Мои группы** → выберите группу
3. Нажмите **Настройки** → **Подписка на канал**
4. Нажмите **Установить ссылку**
5. Отправьте боту ссылку на канал (например: `https://t.me/yourchannel`)

### Кнопки в Mini App (вкладка «Канал»)

- **📢 Подписаться** — переходит по ссылке-приглашению
- **🔍 Проверить подписку** — проверяет статус подписки

## 🤖 Mini App (веб-интерфейс)

### Запуск Mini App сервера

Mini App запускается вместе с ботом. Для публичного доступа используйте ngrok:

```bash
# Установите ngrok: https://ngrok.com
ngrok http 8765
```

Полученный URL вставьте в `.env`:
```env
WEBAPP_URL=https://ваш-url.ngrok-free.app
```

### Вкладки Mini App

| Вкладка | Содержимое |

| 📊 Панель | Статистика, последние нарушения, статус бота |

В @BotFather: `/setmenubutton` → укажите URL вашего Mini App

## ⚙️ Панель управления (в боте)

Все настройки доступны через личные сообщения с ботом:

```
/start → Мои группы → [выберите группу] → Настройки
```

### Что можно настроить

- ⏱ **Время мутов/киков** — длительность наказаний
- 🧠 **ИИ-модерация** — вкл/выкл
- 📨 **Лимит спама** — количество сообщений в минуту
- 🚫 **Запрещённые слова** — список слов для немедленной реакции
- ✅ **Слова-исключения** — слова, на которые бот не реагирует
- 📋 **Правила вкл/выкл** — отдельные категории нарушений
- ⛔ **Автобан** — автоматический бан за тяжёлые нарушения
- 📝 **Режим логирования** — full / minimal / off
- 📌 **Время закрепления** — сколько висит предупреждение
- 🔔 **Подписка на канал** — ссылка для обязательной подписки

## 🛡 Как работает модерация

### Приоритеты проверки (по порядку)

1. **Запрещённые слова** → немедленный мут (без ИИ, мгновенно)
2. **Антиспам** → проверка флуда, дублей, эмодзи
3. **ИИ-анализ** → сравнение с правилами из `правила.txt`

### Шкала наказаний

```
⚠️ Предупреждение → 🔇 Мут → 🚪 Кик → ⛔ Бан
```

- Угрозы и насилие → **немедленный бан** без предупреждений
- Тяжёлые нарушения (severity 5) → **немедленный бан**

### Режим ЧС

Активируется в панели группы. Снижает лимиты спама (5 сообщений вместо 15).

## ❓ Частые вопросы

**Q: Нужно ли перезапускать бота после изменения правила.txt?**
A: Нет. Бот автоматически подхватывает изменения.

**Q: Бот не реагирует на нарушения?**
A: Проверьте: 1) бот — администратор группы, 2) ИИ-ключ задан и корректен, 3) `правила.txt` существует.

**Q: Как добавить бота в несколько групп?**
A: Просто добавьте бота в каждую группу. Настройки для каждой группы — независимые.

**Q: Как посмотреть логи?**
A: `/start` → Логи, или проверьте файл `bot/logs/bot.log`.
